LLM Gateway Demo

Multi-provider LLM gateway with streaming, ephemeral tokens, and tenant routing.

1. Quick Start

Click a demo key below to auto-fill, then hit Connect:

acme-secret-key-123 gpt-4.1-mini globex-secret-key-456 gpt-4.1-mini codex-secret-key-789 o4-mini

2. Supported Providers

openai       GPT-4o, GPT-4o-mini, o1 ...
anthropic    Claude 4, Sonnet, Haiku ...
google       Gemini 2.0, 1.5 Pro ...
openai-compat Groq, Mistral, DeepSeek,
               Ollama, Together, vLLM ...

3. Add a Tenant

// tenants.json
{
  "tenant-myapp": {
    "name": "My App",
    "apiKey": "my-secret-key",
    "provider": "anthropic",
    "model": "claude-sonnet-4-20250514",
    "systemPrompt": "You are...",
    "maxTokens": 2048,
    "rateLimit": 60
  }
}

4. Connect From Your App

// Step 1: Get WS token
POST /api/token
Headers: { "x-api-key": "my-secret-key" }

// Step 2: Open WebSocket
ws://host:3000/ws?token=<token>

// Step 3: Send message
{ "type": "message",
  "content": "Hello!" }

// Receive streaming chunks:
{ "type": "chunk", "content": "Hi" }
{ "type": "done",  "usage": {...} }

5. Initial Context (preload chat history)

Pass existing conversation history when creating the token — the LLM sees it all from the start:

POST /api/token
{
  "initialContext": "Team chat for Project Alpha.\nMembers: Sasa (lead), Marko (backend).\nStack: Angular + NestJS, PostgreSQL, Stripe.",

  "initialMessages": [
    { "role": "user", "content": "Marko: Cart API is pushed" },
    { "role": "user", "content": "Ana: Frontend cart is ready" },
    { "role": "user", "content": "Sasa: Let's handle empty cart edge cases" }
  ]
}

The LLM receives the full history on every message — no need to resend it each time.

6. Per-message Context (runtime metadata)

Attach metadata to any message — user info, current page, selected items, etc.:

// Send via WebSocket
{
  "type": "message",
  "content": "What can you tell me about this order?",
  "context": {
    "userId": "user-42",
    "userName": "Sasa",
    "currentPage": "/orders/12345",
    "selectedOrder": {
      "id": "ORD-12345",
      "status": "shipped",
      "total": 1299.99
    }
  }
}

Context is injected per-message — send different context each time or omit it entirely.
